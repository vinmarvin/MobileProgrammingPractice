Common mistakes

No JOINs exist

Store IDs as strings. Make two separate queries. Accept two reads instead of one joined query.

Subcollection walls

You cannot query a subcollection across different parent documents. If cross-parent queries might happen later, use root from the start.

Duplication is intentional

In SQL duplication is a mistake. In Firestore, storing authorName on each tweet to avoid a second read is correct design.

Design backwards from the screen

SQL: schema first, queries second. Firestore: look at what each screen needs to display, then build the model around those reads.
